# UnitySOP
## What is this project
This project integrates a basic workflow for setting up a development environment for those starting in Unity, specifically with organizational use in mind. This project should be used as a baseline for setting up a development workflow and does not apply to specialized tools outside of the Unity editor and main project repository.
## How to navigate this repo
### Artifacts
Holds screenshots featured in the SOP docx file, reproducible through navigating Unity.com, Unity Hub, GitHub and GitHub Desktop to the step featured in the SOP and using your devices screenshot feature
### Diagrams
Diagrams are seperated into BPMN and Mermaid diagram folders, each containing a raw code file and visual output. Diagrams can be regenerated by manually organizing SOP steps verbatim into the respective visual format, or by following the propmpt log and manual editing documentation found withing the prompt-log folder using the same Gemini CLI model
### Docs
Folder containing the SOP in a word document format accompanied by images. This document can be regenerated through use of the markdown version found in the root folder.
### Prompt-Log
Folder containing full prompt logs used for any AI-generated or assisted material in the repository for reproducability.
### SOT and generated files
The source of truth files are:
- The SOP versions, both .docx and .md
- The glossary derived from the SOP files
- Artifacts
- Prompt Logs

Generated files are
- Diagrams

### Versioning
All changes and version history can be found within both SOP files.
